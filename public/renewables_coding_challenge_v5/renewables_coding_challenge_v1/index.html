Welcome to Avathon Full Stack Coding Challenge!!

Avathon Renewable Suite is a predictive analytics and asset management platform for renewable energy. You can learn more about it at https://avathon.com/industries/energy/renewables/. Our platform gets large volumes of live data from turbines from multiple windfarms which needs to be displayed in user friendly format for analysis by field technicians. Your goal is to primarly use React along with any other open-source or free library as you see fit (we use ag-grid and highcharts) to build a single page dashboard/analytics application to display the attached sample data.

The challenge folder contains device.json file that has list of devices across two windfarms named Mineapollis and Colorado. There is another file fault.json that contains the details of fault events that have happened over time across these devices. Your goal is to display this information in charts and table as follows:

  • Page Level Filter - Dropdown to select between Mineapollis and Colorado Windfarms
  • Additional Filters - Additional filters for users to select specific Device Name, Time Range, Fault Type, Code
  • Tiles - Show critical information like count, duration etc of filtered alarms
  • Top 10 Alarms Codes (Duration) - Bar Chart to show top 10 alarms codes by Duration
  • Top 10 Alarms Codes (Frequency) - Bar Chart to show top 10 alarms codes by Frequency
  • Alarms by Category (Duration) - Pie Chart to show the Alarm Duration grouped by Category
  • Alarms by Category (Frequency) - Pie Chart to show the Alarm Duration grouped by Category
  • Display data in a table - Show all entries in a user friendly table and to allow for sorting and filtering by date, duration, alarm code, description etc.
Bonus: Add page level filter to filter charts for each turbine; Add heat maps for Top 10 alarms and you can containerize the app if you want to go above and beyond!

If applying for senior roles or with experience please ensure proper code structure, documentation, test automation and any additional elements that will showcase your architecture capabilties.

There are no right or wrong answers here but what matters is user experience when they view the large amounts of information. We have attached a sample mockup image as inspiration. This is just for reference and not something we expect as is. We would love to see something even more awesome and better for users. Add a readme if you wish to share any additional information or steps to run the application, notes etc. If requirements seem unclear then document your assumptions. Notes on your design choices will help us understand your thought process.
